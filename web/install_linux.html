Installing Bauble on Linux

Disclaimer: This doesn't describe how to setup and install a database server. If you don't know how to do that then you can use SQLite as the database backend to Bauble.

SQLite is a file based database and though it won't have the same features as the other databases such as networking and multiuser support it may be all you need to manager your collections.

The Installation

1. Install gtk and pygtk using the packaging tool(apt, rpm, emerge, etc..) for your distribution whether it might be.

2. Install SQLObject and FormEncode. Your distribution might not include this in their repository. If you're not sure then install the python setuptools by downloading and running the ez_setup.py script and then at the command line type 'easy_install SQLObject' and FormEncode should be installed automatically.

3. Choose a database engine. Download and install the database engine and it's python connector. These can most likely be installed using the packaging tools for your distribution of choice. Bauble has been tested with Postgres 8.0, MySQL and SQLite though in theory all the database listed below should work.

Database Connector
SQLite SQLite connector
MySQL mysql-python connector
Postgres psycopg connector
Firebird/Interbase KInterbaseDB connector
SapDB/MaxDB sapdb connector
Sybase Sybase connector

If you are using one of the above databases other than SQLite then you will need to create an empty database on the server and create a user to login to the database. You will probably also need to grant the user name you created the permission to modify the database.

4. Download Bauble and extract the archive. From the command line go to the directory where you extracted the files and type 'sudo python setup.py install'.

5. Type bauble at the command line to run Bauble.
